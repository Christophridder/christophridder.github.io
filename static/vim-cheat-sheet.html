Vim cheat sheet_

Vims styrke er ikke et hav af genveje, man skal udenad — det er en grammatik. Du sætter en handling sammen med et mål, og kombinationerne giver sig selv. Lær reglen, ikke listen.

Grammatikken

Handling
y · d · c
yank · delete · change
+
Antal (valgfrit)
2 · 3 · 10
hvor mange gange
+
Mål
w · $ · i{
bevægelse el. text object

Læs det som en sætning: d + 2 + w = "slet to ord". c + i{ = "skift alt inde i de krøllede parenteser". y + $ = "kopiér til linjens slutning". Samme tre operatorer, uendeligt mange mål.

Modes — hvor er jeg?

Esctilbage til normal mode (navigér, kommandér)

ainsert — skriv efter markøren
ony linje under + insert
Ony linje over + insert
vvisual — markér tegn
Vvisual line — markér hele linjer
Ctrl-vvisual block — markér en kolonne
:command — gem, luk, søg/erstat

Visual mode — sådan bruges det

vtænd markering, og flyt så markøren (h j k l, w, $ …)
ykopiér det markerede
dslet (klip) det markerede
cslet markering + skriv noget nyt
> <ryk markering ind / ud
ohop til den anden ende af markeringen
gvgenvælg sidste markering

Tankegang: v tænder, du maler hen over teksten med en bevægelse, og trykker så y/d/c. Visual er "se det først, handl bagefter".

De tre operatorer

yyank — kopiér (bliver, sletter ikke)
ddelete — slet (klip ud)
cchange — slet og sæt mig i insert
yydobbelt = hele linjen → dd, cc samme mønster

c er den snedige: den sletter og lader dig skrive med det samme. d sletter og efterlader dig i normal mode.

Text objects — i vs a

iwinner word — ordet selv
awa word — ordet + mellemrum
i{alt inde i { } (uden parenteser)
a{alt inkl. { }
i" i(inde i anførselstegn · runde parenteser
itinde i et HTML-tag (dine <img>!)

Markøren skal bare være et sted indenfor blokken — vim finder selv de omsluttende tegn. i = inner, a = around.

Yank i praksis

yyén linje · 3yy = tre linjer
y$fra markør til linjens slut
y0 y^til linjestart · til første tegn
ylét tegn · y3l, y10l = flere
ywtil næste ord · y2w = to ord
yetil ordets slutning
yi{alt inde i { } · ya{ = inkl. parenteser
p Pindsæt efter · før markøren

Change-familien

cwskift ord (til ordets slut)
ciwskift hele ordet, uanset hvor i det du står
ccskift hele linjen (beholder indrykning)
c$skift til linjens slut
ci{skift alt i { } — fx en f-string-blok
ci"omskriv en hel streng på én gang
citskift indhold i et HTML-tag

Står du i f"{R_i_maalt:.3g}": ci{ tømmer R_i_maalt:.3g og lader dig skrive nyt med det samme.

Replace: r vs R

roverskriv ét tegn — fx r så x
Rreplace mode — overskriv løbende, tegn for tegn, til du trykker Esc
~skift store/små bogstaver på tegnet
xslet ét tegn (uden insert)

r = "ret lige det ene tegn". R = "skriv hen over som en overskriv-tilstand" — tænk gammeldags skrivemaskine.

System-clipboard

"+yyank til systemets udklipsholder (plus-register)
"+yyhele linjen til clipboard
"+yi{indhold i { } direkte i clipboard
"+pindsæt fra systemets clipboard

Register "+ skal stå før operatoren. Vil du slippe for det hver gang: sæt "vim.useSystemClipboard": true i settings.json — så ryger alle yanks automatisk i clipboard.

Snedige tricks

*søg efter ordet under markøren (n = næste)

Jslå denne linje sammen med den næste
ddpbyt to linjer om · xp = byt to tegn

Workflow-guld: lav én ændring med ciw, hop til næste forekomst med n (efter *), og tryk . for at gentage. "Søg → ret → punktum → punktum".

Gem & luk

:wgem
:qluk
:wqgem og luk · :x gør det samme
:q!luk uden at gemme (kassér ændringer)
:%s/gl/ny/gfind og erstat i hele filen

I VSCodium gemmer Cmd/Ctrl+S også fint — vim-kommandoerne er bare hurtige, når fingrene allerede er på tastaturet.
